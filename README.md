＜開発のバックグラウンドと課題＞
　現在、約100人規模の軽音部では、スタジオ管理にGoogleカレンダーを使用しているが、次のような課題がある。
　・同期タイムラグ：ある端末から入力してから他の端末の反映までに10分以上の遅延が起こり、ダブルブッキングの原因となる。
　・アカウント制限：Googleアカウントのセキュリティの観点から、約100人規模の同時アクセスができず、一部の部員が使用できない状態にある。
　また、スタジオを予約する理由として主に、バンド練習、個人練習の2つの目的から成るため、これらを効率よく部員で使えるよう、Googleカレンダーにはない追加機能を実装したいと考えた。
　仕様として、約1か月に一度の頻度で開催されるライブに対し、バンド練習としての予約はライブ開催日の一か月前から当日まで可能とし、個人練習を目的とした予約は、開催日の2週間前からライブ1週間後まで可能としたい。これは2025年、2026年における軽音部のミーティング、ライブ日程、出演バンド応募締切日のサイクルを参考にした。
　また、個人練習の予約にあたって、効率を考え、パート別で予約できるシステムを構築できるようにした。
＜技術スタック＞
　・バックエンド：Python(FastAPI)
        複雑な予約バリデーションを型安全かつ高速に処理できるため
　・フロントエンド：React(Next.js)
        100人がスマホから快適に操作でき、SPA(Single Page Application)を実現するため
　・データベース：Supabase(PostgreSQL / Realtime)
        WebSocketによるリアルタイム更新を最小限の工数で実装、遅延を解消するため
　・エクスターナルAPI：Google Calender API
